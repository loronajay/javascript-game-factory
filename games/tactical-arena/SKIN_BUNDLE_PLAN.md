# Skin Bundle Plan

This document is the storefront source of truth for grouping the currently
authored skins into packs, exclusive pack extras, small theme packs, and
individual skins. The game can keep everything unlocked for testing, but future
ownership should follow these categories.

Current catalog baseline from this plan: 108 sellable/displayed skins, plus 3
planned Arcane skins. Ghoul skins exist in the asset manifest, but are only
listed in storefront packs when an authored pack explicitly includes them.

## Future Implementation Notes

- Keep `src/ui/skinManifest.generated.js` generated from files only. Do not put
  pricing, rarity, pack membership, or availability in the generated manifest.
- Add an authored catalog later, for example `src/ui/skinCatalog.js`, with pack
  ids, availability, and exact `{ type, slug }` skin refs.
- Test/dev mode can keep everything unlocked while production checks player
  ownership.
- Packs should reference exact unit plus slug pairs, not slug alone.
- Classic/base looks should remain free forever.
